# Remaku v1.0.0 cleanup notes

This file tracks compatibility code and repository conventions that can be removed when Remaku reaches v1.0.0.

## Remaku app

### Remove legacy template metadata migration

- Remove `templates_migrated` from the general config model.
- Remove startup migration calls that scan `Documents/remaku/templates/<macro_id>/*.json`.
- Remove `remaku/services/migration.py` if no other migration uses it.
- Remove tests that only cover legacy template metadata migration.

Why: template capture metadata now lives inside each macro's `templates` section, not in sidecar `templates/*.json` files.

### Remove legacy template metadata import

- Stop reading `templates/<template_id>.json` from imported macro ZIP files.
- Remove the code that merges legacy sidecar metadata into `macro.json`.
- Remove import tests that build archives with `templates/*.json`.

Why: exported and packed macros should include template metadata directly in `macro.json`.

## Compatibility checks before deleting

- Confirm no user-facing import/export path still creates or depends on `templates/*.json`.
- Run `make check-all` in `remaku`.
