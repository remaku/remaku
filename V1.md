# Remaku v1.0.0 cleanup notes

This file tracks compatibility code and repository conventions that can be removed when Remaku reaches v1.0.0.

## Remaku app

### Remove legacy template metadata migration

- Remove `templates_migrated` from the general config model.
- Remove startup migration calls that scan `Documents/remaku/templates/<macro_id>/*.json`.
- Remove `remaku/services/migration.py` if no other migration uses it.
- Remove tests that only cover legacy template metadata migration.

Why: template capture metadata now lives inside each macro's `templates` section, not in sidecar `templates/*.json` files.

### Remove legacy template metadata import

- Stop reading `templates/<template_id>.json` from imported macro ZIP files.
- Remove the code that merges legacy sidecar metadata into `macro.json`.
- Remove import tests that build archives with `templates/*.json`.

Why: exported and packed macros should include template metadata directly in `macro.json`.

### Remove legacy pack catalog fallback

- Remove `PackCatalogEntry.assets` as the download source for Pack Explorer imports.
- Require `default_language` and `language_assets` for official pack entries.
- Remove fallback logic that imports from `assets.zip_url` when `language_assets` is missing.
- Remove legacy pack tests that omit `language_assets`.

Why: pack downloads should be language-specific, selected from `language_assets`.

## remaku-packs

### Remove `packs[].assets`

- Remove `assets` from each `catalog.json` pack entry.
- Keep language-specific download URLs only under `language_assets`.
- Update README examples so they no longer mention `assets.zip_url` as the old-version compatibility URL.

Why: `assets.zip_url` exists only to let older Remaku builds import the default-language ZIP.

### Remove legacy pack folder support

- Remove pack script support for `packs/<game>/<pack>/macro.json`.
- Require language folders such as `packs/<game>/<pack>/zh_TW/macro.json`.
- Remove the legacy GitHub Actions branch that builds `{game}-{pack}.zip`.

Why: all official packs should be packaged as `{game}-{pack}-{language}.zip`.

## Compatibility checks before deleting

- Confirm the latest supported Remaku release can read `language_assets`.
- Confirm official `remaku-packs` releases no longer need to support Remaku builds that only read `assets.zip_url`.
- Confirm no user-facing import/export path still creates or depends on `templates/*.json`.
- Run `make check-all` in `remaku`.
- Run the pack build in `remaku-packs` and inspect at least one ZIP for root-level `macro.json` and `templates/`.
